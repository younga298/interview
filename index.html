project PROCEDURE

메뉴 구조는 CSS와 JS를 적절히 분리해 구현했습니다. hover·focus로 처리 가능한 영역은 CSS로, 포커스가 벗어나도 유지되어야 하는 경우는 JS로 classList.toggle()을 활용해 제어했습니다.

서브페이지의 메뉴 토글 기능은 클릭 이벤트 충돌을 방지했고, 서브메뉴가 있는 항목 클릭 시에만 메뉴가 열리고 닫히도록 설정했고, 내부 항목(서브메뉴 항목)을 클릭했을 때는 메뉴 토글이 되지 않도록 return 처리를 넣어 이벤트를 정교하게 분리했습니다.

Swiper는 breakpoints로 반응형 슬라이드 수를 조절하고, resize 이벤트에 swiper.update()를 적용해 실시간 반영되도록 했습니다.

페이지네이션 위치 문제는 Swiper 기본 스타일을 확인해 position: static으로 재정의해 깔끔하게 정렬했습니다.

레이아웃 전반에는 min-height, rem 단위를 활용해 폰트와 여백이 디바이스 해상도에 따라 자연스럽게 조정되도록 했습니다.

project

제작인원 :1명

기여도 100%

SKILL

HTML, CSS, jQuery, Js

project PROCEDURE

Grid와 Flexbox를 병행해 전체 레이아웃을 유연하게 설계하고, 주요 콘텐츠는 4열 그리드로 균일하게 배치했습니다.

vw 단위를 적극 사용하여 도형, 이미지, 텍스트가 화면 크기에 따라 자연스럽게 크기와 위치를 유지하도록 설계했습니다.

썸네일은 flex: 0 0 calc(100% / 3.5) 와 max-width를 함께 사용해 해상도에 따라 5개 → 3.5개로 유동적 표시가 가능하도록 구성했습니다.

텍스트는 min-width: fit-content, text-overflow: ellipsis, -webkit-line-clamp 등으로 가독성과 레이아웃 안정성을 유지했습니다.

반응형 메뉴는 PC에선 hover, 모바일에선 click으로 구분 처리했으며, 창 크기 변화에 따라 이벤트를 초기화하고 재바인딩하여 오류 없이 작동합니다.

.off()를 활용해 이벤트 충돌을 방지하고, 모바일 메뉴를 열였을때 뒤로가기 까지 잘 작동하도록 직관적으로 개선했습니다.

전체적으로 디바이스에 따라 레이아웃과 기능이 자연스럽게 바뀌도록 반응형 구조를 구현했습니다.
